# i18n Design

## Objetivo

Sistema de internacionalizacion casero para la UI de Sadhana, disenado multi-idioma desde el inicio. Alcance de implementacion (fase B): infraestructura + es/en completos + pali parcial (easter egg).

## Decisiones cerradas

- Enfoque casero (`LanguageContext` + `useT()` + diccionarios), no `react-i18next`. App pequena, 2 idiomas activos; migracion futura barata porque la firma `t('clave')` coincide con la de react-i18next.
- Disenado multi-idioma: diccionarios indexados por codigo de idioma + selector tipo lista (no toggle binario). Anadir un idioma no toca componentes.
- Pali es un locale mas, parcial, easter egg. Romanizado IAST (latino con diacriticos), no Devanagari.
- Una sola arquitectura i18n. Se descarto la "capa de terminologia contemplativa" separada.
- Datos del usuario (presets creados por el usuario, nombres que el usuario teclea) NO se traducen: son contenido en su storage.
- Presets por defecto SI se traducen: son contenido de la app, no datos del usuario. Se resuelven por `t()` via id estable en el momento de mostrarlos. Ver "Presets por defecto".
- Categorias de sonido SI se traducen; nombres individuales de sonido NO. Ver "Sonidos".
- Fechas, duraciones y plurales se resuelven con `Intl` (locale-aware), no con texto preformateado ni reglas a mano.

## Arquitectura de archivos

```
src/i18n/
  index.js              registro de idiomas + metadata (codigo, nombre nativo, dir, script)
  LanguageContext.jsx   provider: estado del idioma, persistencia, gestion de <html lang/dir>, document.title
  useT.js               hook que devuelve la funcion t()
  format.js             helpers locale-aware: formatDuration, formatRelativeDate (via Intl)
  locales/
    es.js               diccionario base (completo, canonico)
    en.js               ingles (completo)
    pi.js               pali (parcial, easter egg)
```

## Diccionarios y claves

- Objetos anidados por dominio, no strings sueltos.
- Uso: `t('modes.practice')`.

```js
// es.js
export default {
  app:     { title: 'Sadhana' },
  modes:   { practice: 'Practicar', design: 'Disenar', remember: 'Recordar' },
  actions: { save: 'Guardar', export: 'Exportar datos', stop: 'Detener audio' },
  panels:  { theme: 'Tema', audio: 'Audio', settings: 'Configuracion' },
}
```

- Interpolacion con reemplazo simple de `{var}`: `t('library.cardMeta', { min: 20 })`. Evita concatenar trozos (orden de palabras y accesibilidad).
- Pluralizacion via `Intl.PluralRules` del idioma activo. Las claves con plural se definen por categoria CLDR:

```js
library: { cueCount: { one: '{n} senal', other: '{n} senales' } }
// t('library.cueCount', { n: 1 }) -> "1 senal"
```

El resolver elige `one`/`other`/... segun `Intl.PluralRules(lang).select(n)`. Resuelve correctamente es/en y cualquier idioma futuro (ru/pl incluidos) sin reglas a mano.

## Formateo locale-aware (Intl)

`src/i18n/format.js`:

- `formatDuration(seconds, lang)`: minutos/segundos legibles segun locale.
- `formatRelativeDate(date, lang)`: via `Intl.RelativeTimeFormat` ("hace 2 dias" / "2 days ago").

`RecentSessions` deja de recibir `when`/`duration` preformateados: recibe el dato crudo (timestamp, segundos) y formatea con estos helpers usando el idioma activo. Fase B cablea es/en; el resto de idiomas lo hereda gratis porque `Intl` ya los soporta.

## Resolucion y fallback

- `t('clave')` busca en el idioma activo.
- Si la clave no existe, cae al idioma base (`es`).
- Permite el easter egg: `pi.js` define solo las claves con termino natural; el resto cae a `es`. Un idioma incompleto nunca muestra clave cruda ni hueco.
- Clave inexistente en activo y en base (typo): devuelve la clave cruda. En desarrollo, ademas `console.warn`. Nunca string vacio (un hueco invisible es peor que una clave fea).

## Sonidos

- Categorias de sonido (grupos): traducibles. `t('sounds.groups.bells')` indexado por nombre de carpeta. Si falta la clave, cae al `groupLabel` actual (carpeta capitalizada), asi una carpeta nueva no rompe nada.
- Nombres individuales de sonido: NO se traducen (demasiados, cercanos al nombre de archivo). Siguen via `labelFromPath`.

## Presets por defecto

- Son contenido de la app, traducibles. Distintos de los presets del usuario.
- No se traduce lo ya sembrado en storage (romperia datos). El storage guarda el `id` estable; la UI resuelve el nombre con `t('presets.<id>')` al mostrarlo.
- Los presets del usuario muestran su `name` literal (no pasan por `t()`).
- Escala: cada preset por defecto nuevo = un `id` estable + sus claves de nombre en cada locale. Sin cambios de codigo en componentes.

## Selector de idioma + persistencia

- Lista (no toggle) en el panel Configuracion de `GlobalPanel.jsx`.
- Cada opcion muestra su nombre nativo (Espanol, English, Pali...).
- Persistido en `localStorage`, igual que tema y acento.
- Primer arranque sin valor guardado: detecta `navigator.language`, hace match por prefijo (`en-US` -> `en`) contra los idiomas registrados, cae a `es` si no hay match.
- Una vez el usuario elige, manda su eleccion guardada.

## Accesibilidad de idioma (incluida en esta sesion)

- El provider actualiza `document.documentElement.lang` y `dir` en un efecto al cambiar idioma.
- El provider actualiza `document.title` desde `t('app.title')`.
- Cada idioma declara su `dir` en metadata (`ar` -> `rtl`, resto `ltr`).
- Se traducen tambien los atributos invisibles: `aria-label`, `title`, `placeholder`.
- Claves como frases completas, nunca trozos concatenados.

## Requisitos anotados (no se implementan en fase B, el sistema los soporta)

- RTL: la gestion de `dir` existe desde el inicio. Al anadir `ar`, el trabajo restante es CSS con propiedades logicas (`margin-inline-start`...). Tarea del idioma arabe.
- Fuentes: font stack que cubra escrituras no latinas (fuentes del sistema o Noto). Decision de cuando se anada el primer idioma no latino.

## Testing

- Resolver de `t()`: navegacion de claves anidadas, fallback al base, interpolacion, pluralizacion (`Intl.PluralRules`), comportamiento ante clave inexistente.
- Integridad: `es` es canonico; cada locale completo (`en`) debe contener todas las claves de `es`. El pali esta exento (parcial por diseno).
- El provider fija `lang`, `dir` y `document.title` correctos al cambiar idioma.
- `format.js`: `formatDuration` y `formatRelativeDate` producen el formato esperado para es/en.

## Fuera de alcance

- Auditoria WCAG completa (aria-pressed/aria-current, aria-live, contraste AA, prefers-reduced-motion, navegacion por teclado del timeline, focus management): sesion separada, registrada en TODO de `state.md`.
- Traduccion de los 14 idiomas no implementados en fase B.
- CSS RTL y carga de fuentes no latinas.
- `meta description` de `index.html` (SEO, no aplica a app local sin indexar).
